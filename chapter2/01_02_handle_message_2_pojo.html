处理微信服务器消息 - 定义消息类对象

我们与微信服务器的交互主要由两个地方，一个是接收消息，一个是回复消息。为了方便操作，我们根据微信官方文档将其抽象为POJO对象，方便我们的开发。

我们把POJO对象分为两类，一类是请求类消息对象，对应我们接收微信服务器的消息。另一类是响应类消息对象，对应我们响应微信服务器的消息。

请求类消息对象

我们接收微信服务器的消息一共有7种类型，分别是：

  • 文本消息
  • 图片消息
  • 语音消息
  • 视频消息
  • 小视频消息
  • 地理位置消息
  • 链接消息

上面7种消息都由一些共同的字段，我们把这些字段提取出来封装成一个基类。

package com.chanshuyi.pojo.message.req;

/**
 * 请求类，消息基类
 *
 * @author chenyr
 * @date 2015.10.03
 */
public class BaseMessage {
    // 开发者微信号
    private String toUserName;
    // 发送方帐号（一个OpenID）
    private String fromUserName;
    // 消息创建时间 （整型）
    private long createTime;
    // 消息类型（text/image/location/link）
    private String msgType;
    // 消息id，64位整型
    private long msgId;

    public String gettoUserName() {
        return toUserName;
    }

    public void settoUserName(String toUserName) {
        toUserName = toUserName;
    }

    public String getfromUserName() {
        return fromUserName;
    }

    public void setfromUserName(String fromUserName) {
        fromUserName = fromUserName;
    }

    public long getcreateTime() {
        return createTime;
    }

    public void setcreateTime(long createTime) {
        createTime = createTime;
    }

    public String getmsgType() {
        return msgType;
    }

    public void setmsgType(String msgType) {
        msgType = msgType;
    }

    public long getmsgId() {
        return msgId;
    }

    public void setmsgId(long msgId) {
        msgId = msgId;
    }
}
